# Costco 健身與減脂共享日誌：Vercel 部署與架構紀錄指引

本專案已完成全部開發，並在 Vercel 生產環境上線！此文件記錄了部署流程、金鑰修復、雲端資料庫建置以及系統的使用指引。

---

## 🔒 1. Gemini API 金鑰與模型修復紀錄

*   **金鑰格式確認**：新註冊的 Google AI Studio 金鑰會以 **`AQ.`** 開頭（傳統金鑰為 `AIzaSy`），此格式完全正常且正確。
*   **模型版本升級**：
    *   由於 `gemini-1.5-flash` 在新註冊專案中預設已不支援呼叫（會回傳 404 NOT_FOUND 錯誤）。
    *   後端代理 API 已成功升級為目前官方最新且具備免費額度的 **`gemini-flash-latest`** 模型（經測試已能 100% 正確完成多模態食材照片及文字分析）。
*   **環境變數設定**：金鑰已安全地託管在 Vercel 專案後台的環境變數 `GEMINI_API_KEY` 中，絕無洩漏給瀏覽器或 GitHub 的風險。

---

## ☁️ 2. Vercel KV (Upstash Redis) 雲端資料庫連結

為實現兩位在不同裝置上**即時同步日誌與歷史體重體脂數據**，我們在 Vercel 專案上綁定了免費的 **Upstash for Redis** 資料庫。
*   **安全環境變數**：連結後，Vercel 會自動注入 `STORAGE_REST_API_URL` 與 `STORAGE_REST_API_TOKEN`，後端 `api/save.js` 與 `api/load.js` 會自動識別並實現數據存取。
*   **狀態指示**：網頁頂部會顯示 **`☁️ 雲端已同步`**，代表兩人的手機與電腦已成功連入同一個雲端資料庫。

## 🎨 3. 新增與優化之核心功能

1.  **🙋‍♂️🙋‍♀️ 獨立體重體脂輸入表單**：
    *   在「雙人每日日誌」中，將原本合併的登錄區拆分為兩個並排的卡片（男生/女生專屬）。
    *   兩位可各自選擇日期並輸入體重、體脂，點擊個別儲存，完全避免數據被對方不小心覆蓋的風險。
2.  **📅 各表單獨立日期選擇器**：
    *   「飲食日誌」與「運動日誌」卡片內新增獨立的日期欄位，匯入或新增日誌更加精確靈活。
3.  **👥 AI 辨識分配選單（一鍵匯入）**：
    *   在 AI 辨識區頂部增加「此食材/照片分配給」選單。辨識完成後，只需點擊一下「將分析食材匯入今日菜單進度」，系統就會自動根據所選的對象與日期，一鍵寫入雲端日誌，**不再彈出任何網頁詢問視窗**。
4.  **📈 雙人減脂折線趨勢圖**：
    *   在「減脂歷史趨勢圖」分頁中，使用 Chart.js 自動渲染出兩人的體重與體脂率下降曲線（男生藍線、女生綠線），讓減脂進度一目了然。
5.  **🔗 一鍵複製共享網址**：
    *   側邊欄提供一鍵複製網址按鈕，方便傳送給伴侶在手機瀏覽器上開啟使用。
